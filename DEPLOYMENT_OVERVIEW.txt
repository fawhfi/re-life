🚀 快速开始 - 3 种方式
================================================================================

方式 1: 一键部署（最简单）
--------------------------------------------------------------------------------
Windows:
  > deploy-all.bat

Mac/Linux:
  $ chmod +x deploy-all.sh
  $ ./deploy-all.sh

方式 2: 按步骤部署（推荐）
--------------------------------------------------------------------------------
1. 阅读文档:
   > cat START_HERE.md

2. 部署后端:
   > cd backend
   > railway login
   > railway up

3. 部署前端:
   > cd frontend
   > deploy-cloudflare.bat  # Windows
   > ./deploy-cloudflare.sh # Mac/Linux

方式 3: 本地测试后部署（最稳妥）
--------------------------------------------------------------------------------
1. 本地测试:
   > test-local.bat         # Windows
   > ./test-local.sh        # Mac/Linux

2. 验证通过后执行部署（见方式 1 或 2）

🎯 推荐部署架构
================================================================================

前端: Cloudflare Pages
  - 成本: $0/月（免费）
  - 特点: 全球 CDN、自动 HTTPS、无限带宽
  - URL: https://relife-app.pages.dev

后端: Railway
  - 成本: $5/月
  - 特点: 支持 Python + ONNX、简单部署、自动扩展
  - URL: https://your-app.railway.app

数据库: Firebase Realtime Database
  - 成本: $0/月（免费额度）
  - 特点: 实时同步、高可用

总成本: $5/月
部署时间: 10 分钟

================================================================================
📋 部署前准备
================================================================================

必需的信息:
  1. Firebase 配置（8 个参数）
     - API Key, Auth Domain, Database URL, Project ID
     - Storage Bucket, Messaging Sender ID, App ID

  2. AI API 密钥（至少一个）
     - NVIDIA API Key
     - OpenAI API Key
     - Google Gemini API Key

  3. Cloudflare 账号
  4. Railway 账号（或 Render/Google Cloud）

可选的信息:
  - SMTP 邮箱（验证码功能）
  - SerpAPI Key（新闻功能）

================================================================================
🐛 常见问题
================================================================================

问题 1: CORS 错误
解决:
  $ railway variables set CORS_ORIGINS="https://relife-app.pages.dev"
  $ railway redeploy

问题 2: 环境变量未生效
解决:
  - 确认前端变量以 VITE_ 开头
  - 在 Cloudflare Pages Dashboard 配置
  - 触发重新部署

问题 3: API 请求失败
解决:
  - 检查 VITE_API_BASE_URL 是否正确
  - 验证后端服务是否运行

问题 4: 构建失败
解决:
  - 本地测试: npm run build
  - 查看构建日志找到具体错误
  - 确认 Node.js 版本 18+

更多问题? 查看 QUICK_DEPLOY.md 的故障排除部分
